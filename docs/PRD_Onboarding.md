# PRD: 온보딩 Feature

> 로아 캘린더 최초 실행 시 사용자를 안내하고 필수 초기 설정을 완료하는 흐름.

---

## 1. 개요

앱 최초 실행 시 권한 요청, 기본 데이터 생성, 주요 기능 소개를 수행한다.
사용자가 최소한의 노력으로 앱을 바로 사용할 수 있도록 한다.

---

## 2. 온보딩 흐름

```
앱 최초 실행
  └─ Step 1: 환영 화면
  └─ Step 2: 기능 소개 (4페이지 스와이프)
  └─ Step 3: 알림 권한 요청
  └─ Step 4: 기본 설정
  └─ Step 5: Apple Focus Mode 안내 (선택)
  └─ Step 6: 완료 → 메인 화면
```

### 2.1 Step 1: 환영 화면

- 앱 로고 + "로아 캘린더" + "일정을 정리하세요"
- "시작하기" 버튼

### 2.2 Step 2: 기능 소개 (스와이프 4페이지)

| 페이지 | 제목 | 설명 |
|--------|------|------|
| 1 | 캘린더 | 구글 캘린더 수준의 일정 관리 |
| 2 | 뽀모도로 | 집중 타이머로 생산성 향상 |
| 3 | Todo | 3가지 날짜 시스템으로 체계적 할 일 관리 |
| 4 | 자기관리 | 통계, 리뷰, 목표로 성장 추적 |

- 각 페이지에 간결한 일러스트 + 한 줄 설명
- 하단 페이지 인디케이터
- "건너뛰기" 버튼 (우상단)

### 2.3 Step 3: 알림 권한 요청

- "알림을 허용하면 일정, 타이머, 마감 알림을 받을 수 있습니다"
- iOS 알림 권한 다이얼로그 표시
- 거부 시에도 진행 가능 (설정에서 나중에 변경 안내)

### 2.4 Step 4: 기본 설정

#### 자동 실행 (사용자 입력 없음)

- 기본 캘린더 자동 생성:
  - "개인" (블루베리, 기본 캘린더)
  - "뽀모도로" (세이지, 세션 자동 기록용)
  - "공휴일" (토마토, 한국 공휴일 읽기 전용)
- SwiftData ModelContainer 초기화 (App Group 포함)

#### 사용자 선택 (선택사항)

| 설정 | 옵션 | 기본값 |
|------|------|--------|
| 첫 요일 | 월요일 / 일요일 | 월요일 |
| 퇴근 시간 | 시간 선택 | 오후 6시 |
| 기본 뽀모도로 프리셋 | 6종 중 선택 | 클래식 (25/5) |

- "나중에 설정할게요" 버튼으로 건너뛰기 가능 (기본값 적용)

### 2.5 Step 5: Apple Focus Mode 안내 (선택)

- "뽀모도로 세션 중 방해금지 모드를 자동으로 켤 수 있습니다"
- "설정하기" → Shortcuts 앱으로 이동 안내
- "나중에" → 건너뛰기

### 2.6 Step 6: 완료

- "준비 완료! 로아 캘린더를 시작하세요"
- "시작" 버튼 → 메인 화면 (캘린더 탭)

---

## 3. 기술 요구사항

### 3.1 온보딩 완료 플래그

- `UserDefaults`에 `hasCompletedOnboarding: Bool` 저장
- true이면 다음 실행부터 온보딩 건너뜀
- 설정에서 "온보딩 다시 보기" 가능

### 3.2 기본 데이터 생성 타이밍

- Step 4에서 기본 캘린더 3개 생성
- 이미 존재하면 중복 생성하지 않음 (앱 재설치 대응)

### 3.3 권한 요청 순서

| 순서 | 권한 | 시점 |
|------|------|------|
| 1 | 알림 (UNUserNotificationCenter) | 온보딩 Step 3 |
| 2 | 사진 (PhotosPicker) | 캘린더 이벤트 첨부 시 최초 요청 |
| 3 | 위치 (MapKit 자동완성) | 이벤트 위치 입력 시 (네트워크만, 권한 불필요) |

- 사진 권한은 온보딩이 아닌 실제 사용 시점에 요청 (iOS 권장 패턴)

---

## 4. 화면 흐름 (전체)

```
앱 실행
  ├─ hasCompletedOnboarding == false
  │   └─ 온보딩 흐름 (Step 1~6)
  │       └─ 완료 → 메인 화면
  └─ hasCompletedOnboarding == true
      └─ 메인 화면 (탭바)
          ├─ 캘린더 (기본 탭)
          ├─ 뽀모도로
          ├─ Todo
          ├─ 자기관리
          └─ 설정
```

## 5. 비기능 요구사항

- CLAUDE.md의 UX 품질 기준 준수 (햅틱, 애니메이션, 다크모드, 접근성)
- 온보딩 전체 소요 시간: 30초 이내 (건너뛰기 시 5초)
- 스와이프 제스처 + 하단 인디케이터 + 건너뛰기 버튼 동시 지원
- VoiceOver 접근성 완벽 지원
