# PRD: 뽀모도로 Feature

> 로아 캘린더의 집중 관리 기능. 다양한 집중 방식과 캘린더/Todo 연동을 제공한다.

---

## 1. 개요

사용자가 집중 세션을 시작, 관리, 기록할 수 있는 뽀모도로 타이머.
6종 프리셋, 오버플로우 모드, 앰비언트 사운드, 세션 전후 기록을 지원하며,
캘린더/Todo Feature와 유기적으로 연동된다.

---

## 2. 타이머 엔진

### 2.1 프리셋 (6종)

| 프리셋 | 집중 | 짧은 휴식 | 긴 휴식 | 용도 |
|--------|------|-----------|---------|------|
| 클래식 뽀모도로 | 25분 | 5분 | 15분 | 기본, 입문자 |
| 숏 스프린트 | 15분 | 5분 | 15분 | 브레인스토밍, 코드 리뷰 |
| 확장 집중 | 50분 | 10분 | 20분 | 딥워크 |
| 52/17 | 52분 | 17분 | - | 연구 기반 최적 리듬 |
| 울트라디안 | 90분 | 20분 | - | 생체리듬 기반 |
| 커스텀 | 자유 | 자유 | 자유 | 사용자 정의 |

### 2.2 긴 휴식 주기

- 기본값: 4회 집중 후 긴 휴식
- 사용자 커스텀 가능 (2~8회)
- 52/17, 울트라디안 프리셋은 긴 휴식 없음

### 2.3 타이머 동작

| 기능 | 설명 |
|------|------|
| 시작 | 프리셋 선택 후 시작 |
| 일시정지 | 세션 중 일시정지/재개 가능 |
| 종료 | 수동 종료 또는 타이머 완료 |
| 오버플로우 | 타이머 종료 후 카운트업으로 전환. 사용자가 준비될 때 수동 종료 |
| 자동 시작 | 세션 → 휴식 → 세션 자동 전환 (토글, 기본 OFF) |
| 스킵 | 휴식 건너뛰기 가능 |

### 2.4 오버플로우 모드

- 설정된 집중 시간이 끝나면 알림 + 진동
- 강제 종료하지 않고 카운트업으로 전환
- 사용자가 자연스럽게 마무리한 후 수동으로 휴식 전환
- 총 집중 시간 = 프리셋 시간 + 오버플로우 시간 (모두 기록)
- 오버플로우 OFF 시: 타이머 완료 → 알림 + 자동으로 휴식 전환 (자동 시작 ON이면 바로 시작, OFF이면 대기)

## 3. 세션 기록

### 3.1 세션 전 (Pre-session)

- "무엇에 집중할 것인가?" 의도 입력 (선택)
- 오늘의 Todo에서 작업 선택 (Todo 연동)
- 프리셋 선택

### 3.2 세션 중

- 타이머 카운트다운 표시
- 현재 세션 번호 / 총 세션 표시 (예: 2/4). 긴 휴식 주기 기준. 긴 휴식 없는 프리셋(52/17, 울트라디안)에서는 미표시
- 선택한 작업 이름 표시
- 앰비언트 사운드 재생

### 3.3 세션 후 (Post-session)

- "무엇을 달성했는가?" 회고 입력 (선택)
- 연결된 Todo 진행률 자동 반영
- 세션 기록 자동 저장

### 3.4 세션 데이터 모델

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 ID |
| cycleID | UUID | 사이클 그룹 ID (4세션+휴식 = 1사이클) |
| sessionNumber | Int | 사이클 내 세션 번호 (1~4) |
| sessionType | SessionType | `.focus` / `.shortBreak` / `.longBreak` |
| preset | PomodoroPreset | 사용한 프리셋 |
| startedAt | Date | 시작 시각 |
| endedAt | Date | 종료 시각 |
| plannedDuration | TimeInterval | 설정 집중 시간 |
| actualDuration | TimeInterval | 순수 집중 시간 = 총 경과 시간 - pausedDuration (오버플로우 포함) |
| overflowDuration | TimeInterval | 오버플로우 시간 |
| intention | String? | 세션 전 의도 |
| reflection | String? | 세션 후 회고 |
| linkedTodoID | UUID? | 연결된 Todo |
| categoryID | UUID? | 카테고리 (PomodoroCategory 참조) |
| pausedDuration | TimeInterval | 총 일시정지 시간 |
| isCompleted | Bool | 정상 완료 여부 (중도 포기 구분) |

### 3.5 추가 데이터 모델

#### PomodoroPresetConfig (커스텀 프리셋 저장)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 ID |
| name | String | 프리셋 이름 |
| focusDuration | TimeInterval | 집중 시간 |
| shortBreakDuration | TimeInterval | 짧은 휴식 |
| longBreakDuration | TimeInterval? | 긴 휴식 (nil이면 없음) |
| longBreakInterval | Int? | 긴 휴식 주기 (nil이면 없음) |
| isDefault | Bool | 기본 프리셋 여부 |

#### PomodoroGoal (주간 목표)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 ID |
| weekStartDate | Date | 주 시작 날짜 |
| targetMinutes | Int | 목표 집중 시간 (분) |
| achievedMinutes | Int | 달성 집중 시간 (분) |

#### PomodoroTimerState (세션 복구용)

| 필드 | 타입 | 설명 |
|------|------|------|
| sessionID | UUID | 진행 중인 세션 ID |
| presetConfig | PomodoroPresetConfig | 사용 중인 프리셋 |
| startedAt | Date | 시작 시각 |
| expectedEndAt | Date | 예상 종료 시각 |
| pausedAt | Date? | 일시정지 시각 (nil이면 진행 중) |
| totalPausedDuration | TimeInterval | 누적 일시정지 시간 |
| currentPhase | SessionType | 현재 단계 (focus/shortBreak/longBreak) |
| sessionNumber | Int | 현재 세션 번호 |

UserDefaults(App Group)에 저장. 앱 종료 후에도 유지.

#### PomodoroCategory (카테고리)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 ID |
| name | String | 카테고리 이름 |
| color | EventColor | 색상 |

세션 데이터 모델의 `category: String?`을 `categoryID: UUID?`로 변경.

- 마지막 사용 사운드 기억

## 6. 통계/분석 (5종)

### 6.1 일간/주간/월간 집중 시간 차트

- 막대 차트로 일별 집중 시간 표시
- 주간/월간 탭 전환
- 오버플로우 시간 별도 색상으로 구분

### 6.2 카테고리별 집중 시간 분포

- 원형 차트 또는 막대 차트
- 카테고리/태그별 집중 시간 비율
- 연결된 Todo 기준으로도 분류

### 6.3 시간대별 생산성 히트맵

- 24시간 x 7요일 히트맵
- 색상 농도로 집중 시간 표시
- 가장 생산적인 시간대 하이라이트

### 6.4 스트릭 추적

- 연속 집중 일수 표시
- 현재 스트릭 / 최장 스트릭
- 하루 최소 1세션 완료(isCompleted = true, sessionType: .focus) 시 스트릭 유지. 중도 포기는 미포함

### 6.5 주간 목표

- 주간 집중 시간 목표 설정 (예: 20시간). 주간 리셋은 캘린더 첫 요일 설정을 따름 (기본: 월요일, 커스텀 가능)
- 진행률 표시 (프로그레스 바)
- 목표 달성 시 축하 알림

## 7. Feature 연동

### 7.1 캘린더 연동

| 기능 | 설명 |
|------|------|
| 세션 자동 기록 | 완료된 세션을 "뽀모도로" 전용 캘린더에 이벤트(EventKind: .event)로 자동 생성. 앱 최초 실행 시 전용 캘린더 자동 생성 |
| 회의 경고 | 세션 시작 전, 향후 N분 이내 캘린더 일정이 있으면 경고 (기본 10분, 설정에서 커스텀) |
| 빈 시간 추천 | 캘린더 빈 시간대에 집중 세션 추천 |

### 7.2 Todo 연동

| 기능 | 설명 |
|------|------|
| 작업 선택 | 세션 시작 시 오늘의 Todo에서 작업 선택 |
| 진행률 반영 | 세션 완료 시 해당 Todo 진행률 자동 업데이트 |
| 예상 뽀모도로 | Todo별 예상 세션 수 설정 (예: 이 작업은 3세션) |
| 완료된 세션 수 | Todo 상세에서 투입 세션 수 확인 가능 |

## 8. Apple 생태계 지원

### 8.1 Live Activity / Dynamic Island

- 캘린더 위젯과 동일한 Widget Extension 타겟에서 구현. App Group으로 데이터 공유
- 세션 중 잠금 화면에 타이머 카운트다운 표시
- Dynamic Island에 남은 시간 + 프리셋 표시
- 일시정지/종료 컨트롤 제공

### 8.2 Apple Watch

- 세션 시작/일시정지/종료 제어
- 현재 세션 카운트다운 표시
- 햅틱으로 세션 종료/휴식 알림
- 컴플리케이션: 오늘 완료 세션 수

### 8.3 위젯

- 홈 화면: 오늘 집중 시간 + 세션 수
- 잠금 화면: 현재 스트릭
- 빠른 시작 위젯: 탭하면 마지막 프리셋으로 즉시 시작

## 9. 백그라운드 동작

### 9.1 포그라운드

- 타이머 카운트다운 실시간 표시
- 앰비언트 사운드 재생
- 화면 꺼짐 방지 옵션 (설정에서 토글)

### 9.2 백그라운드

- 세션 시작 시 종료 시각을 저장, 앱 복귀 시 남은 시간 재계산
- 로컬 알림으로 세션 종료/휴식 종료 안내
- Live Activity로 잠금 화면에 상태 표시
- 앰비언트 사운드는 백그라운드 오디오 세션으로 유지

### 9.3 앱 종료

- 로컬 알림으로 세션 종료/휴식 종료 알림
- 다음 앱 실행 시 미완료 세션 복구: 모달로 "진행 중인 세션이 있습니다. 이어서 하시겠습니까?" 표시
  - "이어서" → 남은 시간으로 타이머 복구
  - "취소" → 중도 포기(isCompleted = false)로 기록

## 10. 알림

| 시점 | 알림 내용 |
|------|----------|
| 세션 종료 | "집중 시간이 끝났습니다. 휴식하세요." |
| 오버플로우 중 | "설정 시간을 초과했습니다. 준비되면 휴식하세요." (1회) |
| 휴식 종료 | "휴식이 끝났습니다. 다시 집중할 준비가 되셨나요?" |
| 긴 휴식 시작 | "4세션 완료! 긴 휴식을 시작하세요." |
| 주간 목표 달성 | "이번 주 목표를 달성했습니다!" |

모든 알림에 햅틱 피드백 포함.

#### 알림 권한 거부 시 대체 UX

- 인앱 배너 + 진동으로 세션 종료/휴식 안내
- 설정 화면에서 "알림을 켜면 백그라운드에서도 알림을 받을 수 있습니다" 안내 표시
- 설정 앱으로 이동 버튼 제공

## 11. 화면 흐름

```
탭바 [뽀모도로]
  └─ 뽀모도로 메인
      ├─ 프리셋 선택
      ├─ 작업 선택 (Todo 연동)
      ├─ 의도 입력 (선택)
      ├─ 시작 → 타이머 화면
      │   ├─ 카운트다운
      │   ├─ 일시정지 / 종료
      │   ├─ 오버플로우 전환
      │   └─ 앰비언트 사운드 제어
      │   ├─ 뒤로 가기 → "세션을 중단하시겠습니까?" 알럿
      │   │   ├─ 확인 → 중도 포기 (isCompleted = false) 기록
      │   │   └─ 취소 → 타이머 계속
      ├─ 세션 완료 → 회고 입력 (선택)
      ├─ 휴식 → 휴식 타이머
      └─ 통계 탭
          ├─ 집중 시간 차트
          ├─ 카테고리별 분포
          ├─ 시간대별 히트맵
          ├─ 스트릭
          └─ 주간 목표
```

## 14. 비기능 요구사항

- CLAUDE.md의 UX 품질 기준 준수 (햅틱, 애니메이션, 3상태 처리, 다크모드, 접근성)
- 세션 날짜 귀속: `startedAt` 기준 (자정 넘어가는 세션도 시작 날짜로 집계)
- 타이머 구현: Date 기반 재계산 방식 (시작 시각 + 경과 시간). Timer 누적 오차 방지
